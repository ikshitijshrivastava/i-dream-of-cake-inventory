Step 1 — Firebase Setup

If you already have Firebase set up from the previous version, skip to Step 2. Otherwise:

  1. Go to https://console.firebase.google.com
  2. Click "Add Project" → Name it i-dream-of-cake
  3. Enable Authentication → Email/Password
  4. Create Firestore Database in asia-south1 region

Step 2 — Update Firestore Security Rules

Go to Firestore → Rules tab → replace everything with:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Click Publish. This already covers the new inventory, grns, and returns collections.

Step 3 — Paste Firebase Config into inventory.html

Important — Do this for BOTH files

Open inventory.html. Find this section (near the bottom):

const firebaseConfig = {
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT_ID.firebaseapp.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT_ID.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID"
};

Replace it with the same config you already use in index.html. Save the file. The two files share the same Firebase project so data syncs automatically.

Step 4 — Upload to GitHub (Step-by-Step)

Method A: GitHub Website (Easiest — No App Needed)
  1. Open https://github.com in Chrome on your phone or tablet
  2. Sign in to your GitHub account
  3. Navigate to your existing repository (the one where index.html already lives)
  4. You will see the list of files already uploaded

Uploading inventory.html (NEW file)

  1. Tap "Add file" button → tap "Upload files"
  2. Tap "choose your files"
  3. Select inventory.html from your phone/tablet storage
  4. Scroll down → in the box that says "Commit changes", type: Add inventory management module
  5. Tap the green "Commit changes" button
  6. Wait a few seconds — the file is now uploaded

Replacing sw.js (UPDATED file)

  1. Tap on sw.js in your file list
  2. Tap the pencil icon (Edit this file) — top right of the file view
  3. Select all the text in the editor (tap and hold → Select All)
  4. Delete it all, then paste the entire new content of sw.js
  5. Scroll down → type commit message: Update service worker for inventory
  6. Tap "Commit changes"

Replacing SETUP.html (UPDATED file)

  1. Same process — tap SETUP.html → pencil icon → replace all content → commit
Method B: GitHub Android App
  1. Install GitHub from Play Store
  2. Open your repository → tap the + button
  3. Tap "Upload file" → select inventory.html
  4. Add commit message → tap Commit
  5. For sw.js, you'll need to edit via the website method above (GitHub app doesn't edit existing files easily)

Step 5 — GitHub Pages Auto-Deploy

Nothing extra to do!

GitHub Pages automatically re-deploys within 1–3 minutes every time you commit a file. After uploading inventory.html, your site will be live at:

https://YOUR-USERNAME.github.io/YOUR-REPO-NAME/inventory.html

The main orders app remains at: /index.html — completely unchanged.

Step 6 — Install / Refresh on Your Devices

On Lenovo Tab P12 or Samsung S24 Ultra

  1. Open Chrome → go to your GitHub Pages URL
  2. If the old app is installed, clear Chrome cache: Chrome → Settings → Privacy → Clear browsing data → Cached images and files
  3. Reload the page — you'll see the updated app
  4. To access Inventory: open https://YOUR-SITE/inventory.html in Chrome → tap 3-dot menu → "Add to Home Screen"
  5. You'll now have two icons: one for Orders, one for Inventory

How the Inventory Module Works

📦 Stock Tab

Add items manually or import from CSV. Each item has: name, SKU, category, location/warehouse, quantity, unit, reorder level, cost, batch number, expiry date, supplier. Filter by location, search anything. Low-stock and expiry alerts appear automatically.

🚚 Receive Tab (GRN)

When goods arrive, tap + Receive Goods. Enter the invoice number, waybill/LR number, supplier, date, and add line items with invoiced vs received quantities. If any quantity doesn't match, the system flags it as a Mismatch and highlights the difference. Stock is automatically updated with received quantities.

↩️ Returns Tab

Log sales returns, purchase returns, or damage claims. Choose whether to deduct from stock automatically. Claim amounts are tracked for supplier recovery.

🔍 Recon Tab

Physical stock count reconciliation. Load items by location, enter physical counts, see differences highlighted. Tap Apply Adjustments to sync system stock to physical reality.

🔔 Reorder Tab

Automatically shows which items need reordering based on their reorder level. Priority-coded (red = urgent, yellow = soon). Shows suggested order quantity and estimated cost.

⬆️ Import / ⬇️ Export

Import bulk inventory via CSV (download the template first). Export full stock list as CSV for records or sharing.
